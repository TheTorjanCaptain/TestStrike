SQL Injection

SQL Injection (SQLi) in penetration testing is the process of exploiting a flaw in a web application’s database query handling. When an application fails to properly validate or sanitize user input, an attacker can inject malicious SQL code into a query. This allows the attacker to manipulate the database, retrieve unauthorized data, alter records, or even execute administrative operations, such as deleting or modifying data, potentially compromising the application’s security.
